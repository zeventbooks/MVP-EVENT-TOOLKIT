üîÑ Development Workflow

Standard processes for daily development, staging, and production releases

Daily Development

# Make code changes, then deploy to @HEAD ./scripts/dev-deploy.sh # Or manually: npm run test:jest # Run unit tests npm run push # Push to Apps Script sleep 30 # Wait for propagation ./use-head-deployment.sh # Verify @HEAD npm run test:newman:smoke # Quick API test

Staging Release

# Prepare staging release (v1.4.0) ./scripts/create-staging.sh 1.4.0 # Create deployment in Apps Script UI, then: ./scripts/tag-staging.sh 1.4.0 <DEPLOYMENT_URL> # Run full test suite on staging npm run test:newman -- -e postman/environments/mvp-event-toolkit-staging.json npm run test:e2e

Production Release

# Promote to production (after QA sign-off) ./scripts/promote-to-production.sh 1.4.0
  • Code changes ‚Üí ./scripts/dev-deploy.sh
  • Git commit and push to branch
  • Create PR ‚Üí Team review ‚Üí Merge to main
  • Weekly staging ‚Üí ./scripts/create-staging.sh
  • Full QA testing on staging
  • Production promotion ‚Üí ./scripts/promote-to-production.sh

üì¶ Version Control & Testing

Code Versioning

  • Jest: Unit + Contract tests

  • Easy rollback capability
